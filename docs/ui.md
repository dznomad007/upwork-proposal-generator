## 컴포넌트별 역할

### ProfileForm (업데이트)
- 상단: 이력서 업로드 영역 (드래그앤드롭 + 클릭)
  - 업로드 중: 스피너 + "이력서 파싱 중..."
  - 캐시 히트: "✓ 캐시에서 불러옴" 초록 배지
  - 신규 파싱: "✓ 이력서에서 불러옴" 배지
  - 에러: 빨간 메시지
- 중단: 스킬 / 경력 / 프로젝트 textarea, 시급 범위
- 하단: "프로필 초기화" 링크 버튼
- autosave 상태: "저장됨" / "저장 중..." 인디케이터

### JobInputSection
- **기본 탭**: 텍스트 붙여넣기 (`textarea`)
- **보조 탭**: URL 입력 → 스크래핑 → 성공 시 텍스트 탭으로 자동 이동
- 스크래핑 실패 시 메시지 표시 후 텍스트 탭 활성화

### JobPreviewCard
- `/api/parse-job` 결과를 카드 형태로 표시
- 제목, 예산, 스킬 태그, 클라이언트 정보
- 누락/불확실 필드를 노란 배너로 표시
- 사용자가 직접 수정 가능한 인라인 편집 지원

### ResultPanel (4탭)

| 탭 | 컴포넌트 | 주요 기능 |
|----|----------|----------|
| 커버레터 | `CoverLetter` | Markdown 렌더링, 복사 버튼, 글자수 표시 |
| 적합도 | `FitScore` | 전체 점수 원형 게이지, 세부 3개 바, 스킬 뱃지 |
| 강조 포인트 | `KeyPoints` | 우선순위(high/medium/low)별 카드 |
| 견적 가이드 | `RateGuide` | 추천 시급, 전략 설명, 근거 텍스트 |

### 공통 하단 영역
- `confidence` 게이지 (0~100)
- `assumptions` 목록: "AI가 가정한 내용"
- `missingInfo` 목록: "제공되지 않은 정보"

## 상태 흐름

```
[앱 로드]
  ↓ Clerk 세션 확인
  ↓ GET /api/profile/active
  → 있음: 프로필 폼 자동 채움
  → 없음: 빈 폼

[이력서 업로드]
  uploading → (캐시 히트) → 즉시 폼 채움
           → (캐시 미스) → POST /api/resumes/upload → 폼 채움 + DB 저장

[프로필 수동 수정]
  입력 → isDirty=true → 500ms debounce → PATCH /api/profile/:id → isSaving=false

[공고 분석]
  idle
    ↓ (공고 텍스트 + 프로필 입력 완료 → 분석 시작 클릭)
  parsing   ← /api/parse-job 호출 중
    ↓
  preview   ← JobPreviewCard 표시, 수정 가능
    ↓ (분석 시작 클릭)
  analyzing ← /api/analyze 호출 중
    ↓
  result    ← ResultPanel 표시
    ↓
  error     ← 실패 시 재시도 버튼 + 에러 메시지
```

## UX 원칙

- 텍스트 붙여넣기 탭이 항상 기본 활성 상태
- URL 스크래핑 실패는 자연스럽게 텍스트 탭으로 전환 (에러가 아닌 폴백)
- 이력서 업로드 후 기존 수동 입력값 덮어쓰기 전 확인 (내용이 있을 경우)
- autosave는 조용히 동작 — 저장 중일 때만 작은 인디케이터 표시
- AI 결과는 "초안"임을 명확히 표시 (편집 유도)
- 로딩 중에는 버튼 비활성화 + 스피너
- 모바일 대응: 입력/결과 영역 세로 배치
