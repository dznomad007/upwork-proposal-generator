# 범위 및 성공 기준

## 포함 (현재 구현됨 ✅ / 구현 예정 🔲)

✅ 공고 텍스트 붙여넣기
✅ 사용자 프로필 텍스트 입력
✅ 공고 구조화 (`/api/parse-job`)
✅ AI 분석 결과 생성 (`/api/analyze`)
✅ 누락 정보 및 가정 표시
✅ 결과 복사 (커버레터, 각 섹션)
✅ AI 프로바이더 전환 (Anthropic / OpenAI — 환경변수)
✅ 입력/출력 Zod 검증
🔲 Clerk 인증 (로그인/회원가입)
🔲 이력서 파일 업로드 (PDF/DOCX → AI 구조화)
🔲 프로필 DB 저장 + 재방문 시 자동 복원
🔲 이력서 중복 업로드 방지 (파일 해시 기반)
🔲 프로필 autosave

## 제외 (향후 단계)

- Cloudflare 우회에 의존하는 자동 스크래핑
- Upwork 자동 로그인 / proposal 자동 제출
- 브라우저 확장 기능
- 여러 프로필 전환 / revision history
- 분석 결과 저장 / 히스토리

## URL 스크래핑 (보조 기능 — 향후 추가)

- Playwright + stealth plugin 사용
- 성공 시 공고 텍스트로 자동 채움
- 실패 시 즉시 텍스트 입력 모드로 전환
- 핵심 흐름과 코드 결합 금지 (별도 모듈)

---

## 성공 기준

1. 스크래핑 없이 텍스트 붙여넣기만으로 분석 결과를 얻을 수 있다
2. 이력서를 한 번 업로드하면 다음 방문 때도 프로필이 유지된다
3. 같은 파일을 재업로드해도 AI API를 다시 호출하지 않는다
4. 부족한 정보와 AI 가정이 명확히 드러난다
5. URL 기능이 실패해도 전체 경험이 깨지지 않는다

## 검증 시나리오

### 공고 분석 (기존)
1. 공고 텍스트 붙여넣기로 구조화 성공
2. 누락 정보가 있을 때 사용자에게 명확히 표시됨
3. 분석 결과가 유효한 JSON으로 반환됨
4. 커버레터 복사가 동작함
5. 빈 입력 / 매우 긴 입력 / 예산 없는 공고를 정상 처리함

### 이력서 + 프로필 (신규)
1. 로그인 후 PDF 업로드 → 프로필 자동 채움 + DB 저장
2. 같은 파일 재업로드 → "이미 업로드한 이력서" 표시, 네트워크 탭에서 AI 미호출 확인
3. 다른 파일 업로드 → AI 새로 호출됨
4. 텍스트 수동 수정 → 500ms 후 자동 저장
5. 새로고침 → 이전 프로필 자동 복원
6. "원본으로 되돌리기" → resume.parsedProfileJson으로 복원
7. 다른 계정 로그인 → 이전 사용자 데이터 접근 불가

### URL 스크래핑 (보조)
1. URL 스크래핑 성공 시 자동 입력이 됨
2. 스크래핑 실패 시 텍스트 입력 모드로 자연스럽게 전환됨
